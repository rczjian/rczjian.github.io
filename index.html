NUS Resonance Website

https://www.nusresonance.com
Next.js
Server Actions
Search Engine Optimisation
In July 2024, I was approached by the incoming Executive Committee to design and build the club website. There was an existing site at the current domain, but the content was extremely outdated - more than 15 years old! The idea was to modernise, update and expand the website's offerings. It would have more public-facing content, to engage potential members, clients and supporters; as well as a members-only portal, for access to exclusive resources and information.

The defining feature of the website is that almost all of the pages can be updated via a Google Sheets spreadsheet - a universal, straightforward and foolproof platform that all students have used before, thereby enabling non-technical contributors to update the site's content regularly and indefinitely with no code at all. To improve performance and avoid issues upstream such as rate-limiting, the spreadsheet content is cached until a site maintainer dispatches a cache purge server action via the in-built admin portal.

I also built the site with responsive design and search engine optimisation in mind in order to boost the club's visibility and marketing reach. This included implementing mobile-first design, server-side rendering, open graph meta tags, JSON-LD structured data and other best practices.

bonkboard.gg

HTML5 Canvas
Next.js
React
Inspired by Reddit's r/place, bonkboard was hacked together in a couple of days during the rise of the meme coin BONK. However, to avoid the combative nature of the pixel wars common in r/place, we designed the board to have 500x500 pixels, such that there was sufficient space for all to paint on. The board data was stored as a 750,000-length uint8 flat array, with each sequence of 3 corresponding to the RGB values of one pixel. Each set of painted pixels submitted was sent as a blockchain transaction, with the user paying a small amount of BONK per pixel painted.

We submitted the project as an entry for the Solana Sandstorm virtual hackathon, where we won 4th place in the Bonking with Bonk! track. Various artists, creators and DAOs from the Solana community have since left their mark on the shared canvas.

Reso Booking System

React
Google Calendar API
As part of concert preparation every year, my university a cappella club, NUS Resonance (i.e. Reso), has to share the sound system (microphones, mixer, speaker) among the numerous concert groups. The booking process used to be a very manual affair, where each and every group I/C had to approach the Resource Manager to request for specific practice timeslots, and the Resource Manager had to then review the requests, work out any conflicts with the relevant I/Cs, and then personally update the club's shared Google Calendar with the finalised bookings.

During my year in the club's Executive Committee, my Resource Manager approached me with the idea to streamline and automate the process with this web application. Group I/Cs could easily submit a booking request with all the required information via the website, and could also see all existing pending and approved bookings, which reduced the number of clashing requests. The Resource Manager could log in via Google SSO to access the admin page, where they would be able to review all the pending requests at once. Automated email updates would be sent upon approval/rejection, and the shared Google Calendar would be updated accordingly too.

Floating Bridge Bot

Python
Telegram Bot API
During the COVID-19 pandemic, many of us moved online for our social interactions - we kept in touch with our friends and family via texts and calls, and had fun together playing games online. Since my friends and I used to play floating bridge in person, I thought to build a Telegram bot that could facilitate the game virtually.

In order to ensure valid and standardised input for the game's backend, I took full advantage of Telegram's custom keyboard and inline keyboard functionalities to provide players with an exhaustive set of all their available input options, while rejecting any other input values. I also sent each player's pre-sorted hand via Telegram's custom keyboard API so they remain a secret to the other players.
